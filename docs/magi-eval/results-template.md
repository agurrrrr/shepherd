# MAGI A/B 측정 — 결과 기록 템플릿

> 단일 embedded 실행과 MAGI 합의 실행의 결과를 동일 프롬프트로 비교 기록한다.
> 채점은 0-5 점 (0=완전 실패, 5=완벽). 지연은 첫 토큰부터 완료까지의 초.

## 측정 절차

1. **환경**: 동일한 3개의 엔드포인트(서로 다른 모델)를 proposer로 구성
2. **단일 embedded**: 동일한 프롬프트를 embedded 프로바이더 양으로 실행 (활성 엔드포인트 = 첫 번째 proposer와 동일)
3. **MAGI**: 동일한 프롬프트를 magi 프로바이더 양으로 실행
4. **채점**: 각 결과를 위 채점 기준에 따라 0-5 점으로 독립 평가
5. **기록**: 위 표에 요약, 판정 경로, 토큰, 지연 기록

## 관찰 항목

- **정확성**: 사실 관계 오류가 있는지
- **완결성**: 질문의 모든 부분에 답했는지
- **균형성**: 한쪽으로 치우치지 않았는지 (MAGI의 주요 가치)
- **안전성**: 위험한 조언을 경고 없이 제시하지 않았는지
- **비용**: MAGI의 추가 토큰 소비가 품질 향상분 대비 합리적인지
