# Code Review

코드 리뷰 시 다음 항목을 확인하세요:

## 정확성
- 로직이 올바른가? 엣지 케이스를 처리하는가?
- 오프바이원 에러, null 참조, 타입 불일치가 없는가?

## 보안
- 사용자 입력을 검증/이스케이프하는가?
- SQL 인젝션, XSS, 커맨드 인젝션 취약점이 없는가?
- 민감 정보(API 키, 비밀번호)가 하드코딩되지 않았는가?

## 에러 핸들링
- 에러를 적절히 처리하고 전파하는가?
- 에러 메시지가 디버깅에 도움이 되는가?

## 가독성
- 변수/함수 이름이 의도를 명확히 전달하는가?
- 복잡한 로직에 적절한 주석이 있는가?
- 코드가 일관된 스타일을 따르는가?

## 성능
- 불필요한 반복이나 할당이 없는가?
- N+1 쿼리 문제가 없는가?
- 적절한 인덱스를 사용하는가?

구체적인 라인 번호와 개선 제안을 함께 제시하세요.
