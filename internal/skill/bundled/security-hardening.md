# Security Hardening

보안 강화 작업 시 다음 항목을 점검하고 수정하세요:

## 인증 및 인가
- JWT 시크릿이 충분히 길고 안전한가? 빈 문자열 방어가 되어 있는가?
- 토큰 만료 시간이 적절하게 설정되어 있는가?
- 인증이 필요한 엔드포인트에 미들웨어가 적용되어 있는가?

## CORS 설정
- 와일드카드(*) 대신 명시적 origin을 사용하는가?
- 허용된 HTTP 메서드와 헤더가 최소 권한으로 설정되어 있는가?
- credentials 허용 시 와일드카드가 아닌 특정 origin을 사용하는가?

## 사용자 입력 검증
- Git 브랜치명, 파일 경로 등에 인젝션 방어가 되어 있는가?
- 쉘 커맨드에 사용자 입력이 직접 전달되지 않는가?
- SQL 쿼리에 파라미터 바인딩을 사용하는가?
- 경로 순회(path traversal) 공격이 방어되는가?

## 내부 정보 보호
- 에러 응답에 스택 트레이스나 내부 경로가 노출되지 않는가?
- 내부 IP, 도메인, 인프라 정보가 클라이언트에 전달되지 않는가?
- 디버그 모드가 프로덕션에서 비활성화되어 있는가?

## 의존성
- 알려진 취약점이 있는 패키지를 사용하고 있지 않은가?
- 최소 권한 원칙으로 의존성이 관리되는가?

수정 시 각 항목에 대해 변경 전/후 코드를 함께 제시하세요.
