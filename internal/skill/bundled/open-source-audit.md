# Open Source Audit

오픈소스 공개 전 다음 항목을 반드시 점검하세요:

## 민감정보 노출
- API 키, 토큰, 비밀번호가 하드코딩되어 있지 않은가?
- JWT 시크릿이 소스코드에 포함되어 있지 않은가?
- 내부 IP 주소, 도메인, 인프라 정보가 노출되지 않는가?
- .env 파일이나 설정 파일에 실제 크리덴셜이 포함되어 있지 않은가?

## Git 히스토리
- 과거 커밋에 민감정보가 남아있지 않은가?
- 커밋 메시지에 내부 정보가 포함되어 있지 않은가?
- 불필요한 브랜치가 정리되었는가?

## 파일 관리
- .gitignore에 빌드 산출물, 환경설정, IDE 파일이 포함되어 있는가?
- 바이너리 파일이 저장소에 포함되어 있지 않은가?
- 불필요한 테스트 데이터나 더미 파일이 정리되었는가?

## 라이선스 및 문서
- LICENSE 파일이 존재하고 저작권 표시가 올바른가?
- README에 프로젝트 설명, 설치 방법, 사용 예시가 포함되어 있는가?
- CONTRIBUTING 가이드가 필요한가?

## 코드 품질
- CORS 설정이 와일드카드(*)가 아닌 명시적 origin으로 설정되어 있는가?
- 디버깅용 코드(console.log, fmt.Println 등)가 제거되었는가?
- 하드코딩된 문자열이 i18n 처리되었는가?

각 항목을 PASS/FAIL로 표시하고, FAIL 항목에 대해 구체적인 파일 경로와 수정 방법을 제시하세요.
