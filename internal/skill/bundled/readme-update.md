# README Update

README를 작성하거나 업데이트할 때 다음 절차를 따르세요:

## 사전 분석
- 현재 코드베이스의 기능과 README 내용 간 차이를 파악
- 새로 추가된 기능, 변경된 설정, 제거된 기능을 확인
- 설치 방법과 의존성이 최신 상태인지 검증

## 필수 섹션
- 프로젝트 개요: 한두 문장으로 프로젝트 목적 설명
- 주요 기능: 핵심 기능 목록
- 설치 방법: 빌드 및 실행 단계
- 사용 예시: 기본적인 사용법 코드/명령어
- 설정 옵션: 환경변수, 설정 파일 설명
- 라이선스: 라이선스 종류 명시

## 스크린샷 관리
- 스크린샷은 assets/ 폴더에 저장
- 더미 데이터로 깨끗한 스크린샷 사용 (민감정보 제거)
- 이미지 경로는 상대 경로 사용

## 스타일 가이드
- 배지(badge)는 상단에 배치
- 목차(TOC)는 섹션이 5개 이상일 때 추가
- 코드 블록에 언어 지정
- 외부 링크는 동작 여부 확인

현재 코드와 일치하지 않는 내용이 있으면 반드시 업데이트하세요.
